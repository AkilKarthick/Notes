in java 8 interfaces can have both default() with default keyword and static() method with static keyword

1.default method : usefuld to include new features in an interface without altering the implementation that exist for the interface
'default ' keyword & it is 'overridable: yes(in implementing class)
default impelemntation which can be redefined.

classes implementing the interface are not forced to override default method

interface TestInterface {
    void square(int a); // abstract method must be overriden by all the class to provide implementation.

    default void show() {  // 
        System.out.println("Default Method Executed");
    }
}

class TestClass implements TestInterface {
    public void square(int a) {
        System.out.println(a * a);
    }

    public static void main(String args[]) {
        TestClass d = new TestClass();
        d.square(4);
        d.show(); // Calls default method
    }
}

output: 

16
Default Method Executed

issues facing during method overloading

when a class implements two or more interface that provdie a default impelemntation for a method with same names and signature, java compiler cannot determine which default impelementation to use, this ambigiuty result in compilation error

we can resolce this 
1. just override the detaul method provid specific implementation to resolve ambiguity
2, explicitly invoke a specific default method, like using super() keyword

interface MyData {

    default void print(String str) {
        if (!isNull(str))
            System.out.println("MyData Print::" + str);
    }

    static boolean isNull(String str) {
        return str == null || "".equals(str);
    }
}

class MyDataImpl implements MyData {
    public static void main(String args[]) {
        MyData.isNull("abc"); // Call static method directly on interface
    }
}
